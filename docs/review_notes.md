# Claude版v1に対する監査結果とv2修正

## 結論

Claudeの方向性は妥当でした。ただし、その説明どおりには実装されていない箇所と、概念を混同した箇所がありました。v1のまま本番運用を始めると、再現性のない局面判定、比較日のずれ、欠損データ混入、予測の事後改変が起き得ます。v2では以下を修正しました。

## 重大な修正

1. **証拠と原因の混同**
   - v1の5等級は「資金流入の証拠の強さ」と「短期ポジション調整という原因仮説」が同じ軸でした。
   - v2は`flow_evidence`（証拠水準・方向）と`move_hypothesis`（原因仮説）に分離しました。

2. **加速・減速の誤判定**
   - 1週・4週・13週リターンは期間が異なるため、単純な大小は加速を意味しません。
   - v2は同じ4週指標の前週差だけを`trend_vs_previous`として使います。

3. **固定ルールなのにGPT判定**
   - v1は局面条件を固定しながら、実際の判定はGPT任せでした。
   - v2はコードが`phase_assessment`を決定し、成立・不成立条件も出力します。GPTは再分類せず、留保だけを示します。

4. **基準日不一致と古いデータの混入**
   - v1は暗号資産の週末価格がSPYの金曜基準日より先に進む可能性がありました。
   - 古いティッカーを`missing`へ記録しても計算から除外していませんでした。
   - v2はSPY基準日後を切り落とし、7暦日超古いデータを計算対象から外します。

5. **予測履歴の保存設計**
   - v1の`judgments`は予測と事後評価の境界が曖昧でした。
   - v2は`predictions`を不変記録、`verifications`を別記録とし、schemaも分離しました。

6. **再現性と鮮度**
   - v2はschema version、run_id、設定hash、完全archive、critical missing、テーマ別coverageを追加しました。
   - Custom GPTは各Phaseでmanifestを再取得し、進行状態行のgeneration IDと照合します。10暦日を超えた古いデータは停止します。

7. **固定テーマの選択バイアス**
   - v1の8テーマは市場全体ではなく、AI・半導体・電力・防衛へ偏っています。
   - v2はこの限界を明記し、登録テーマに対応しない強い業種ETFを`unmapped_positive_industry_signals`として出します。
   - 完全な新テーマ自動発見ではありません。テーマallowlistの四半期レビューは残ります。

## 未解決・本番前に必要なこと

- `data/themes.json`の構成銘柄とroleは暫定です。企業の事業構成、上場状態、重複所属を一次資料で確認する必要があります。
- 実データ取得はこの作業環境では未実行です。疑似価格による全工程テストは通過していますが、GitHub Actions上でyfinanceの実取得を1回確認してください。
- 初回は前週履歴がないため「拡散」は原則として判定不能です。少なくとも2回目以降に同期間比較が始まります。
- 価格・出来高だけでは真のテーマ資金フローを測れません。第1期の上限は「間接証拠」です。
- 閾値は半年〜1年の履歴が貯まるまで最適化しないでください。先に再現性、欠損時停止、撤回規律を検証します。
